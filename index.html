About

I started programming by building things I wanted to use.

How it started

In June 2025, I picked up HTML and JavaScript the way most people start: building small tools and games just to see if I could. There was no plan for an "ecosystem" yet, just curiosity and a habit of finishing what I started.

By November 2025 I'd moved into Python, drawn to how fast it let an idea become something that actually ran. Scripts and automations came first, then backend logic, then full applications.

How it became Wizard

On March 2, 2026, I set out to build an AI chatbot, nothing more ambitious than that at first. It started as basic Python pattern-matching against a learned_phrases.txt file, then a real model running locally through Ollama, served on localhost with Flask. Mid-March, I moved it onto Groq's API and put it fully online through GitHub and PythonAnywhere.

" What if there was more?

The "ecosystem" idea didn't arrive until April 21, the moment I asked myself that question and started sketching what else could plug into the same account, the same login, the same design language. That's where a single chatbot project became Wizard Ecosystem: an assistant, a browser, a search engine, email, notes, a calendar, chat, automation, and a developer API, all sharing one codebase.

Every piece of it is mine — the Flask backend running fifty-plus routes, the eleven-table SQLite schema underneath, the Electron builds for Windows, macOS, and Linux, and the deployment pipeline across PythonAnywhere and GitHub Pages. Architecture, code, and shipping, all while still in middle school.

Beyond the code

Off-screen, I hold a First-Degree Black Belt in Taekwondo, earned in April 2026 through ATA's Leadership and Life Skills programs. I play violin in my school orchestra and compete in MATHCOUNTS and MathLeague. None of it feels separate to me — the discipline that earns a black belt is the same discipline that keeps a nine-application platform running, tested, and maintained.

Architecture & Infrastructure

A Flask backend handles fifty-plus routes across an eleven-table SQLite schema — users, chats, memory, calendar events, uploaded files, API keys, personalities, agents, chains, email aliases, and conversations — tied together through a shared profile system.

The frontend ships two ways: static pages on GitHub Pages for the browser experience, and native Electron installers — NSIS for Windows, DMG for macOS, AppImage for Linux — with auto-updates through GitHub Releases. Streaming responses run over Server-Sent Events, sessions use HTTP-only, SameSite=None, Secure cookies, and CORS stays scoped to the ecosystem's own domains.
