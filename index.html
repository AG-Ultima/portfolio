About

I started programming by building things I wanted to use.

How it started

In June 2025, I began learning HTML and JavaScript the same way most people start: by building small tools and games for myself, just to see if I could. There was no grand plan for an "ecosystem" at that point — just curiosity, and a habit of finishing what I started.

By November 2025, I'd moved into Python, drawn to how quickly it let me turn an idea into something that actually ran. I started building small scripts and automations, then backend logic, then full applications.

How it became Wizard

In March 2026, those separate skills came together into Wizard Ecosystem — a single, unified platform rather than another one-off project. I didn't want to build "an AI chatbot." I wanted to build the full stack of software a person actually uses in a day: an assistant, a browser, a search engine, email, notes, a calendar, chat, automation, and a developer API — all sharing one account, one design language, and one codebase.

Every part of it is mine: the Flask backend with over fifty routes, the eleven-table SQLite schema, the Electron desktop builds for Windows, macOS, and Linux, and the deployment pipeline across PythonAnywhere and GitHub Pages. I designed the architecture, wrote the code, and shipped it to real users — while still in middle school.

Beyond the code

Outside of software, I hold a First-Degree Black Belt in Taekwondo, earned in April 2026 through ATA's Leadership and Life Skills programs. I play violin in my school orchestra, and I compete in MATHCOUNTS and MathLeague. None of these are separate from how I build software — the same discipline that gets a black belt is what gets a nine-application platform actually shipped, tested, and maintained.

Architecture & Infrastructure

A Flask backend handles over fifty routes across an eleven-table SQLite schema — users, chats, memory, calendar events, uploaded files, API keys, personalities, agents, chains, email aliases, and conversations — all tied together through a shared profile system.

The frontend ships two ways: as static pages on GitHub Pages for the browser experience, and as native Electron installers — NSIS for Windows, DMG for macOS, and AppImage for Linux — with auto-updates through GitHub Releases. Streaming responses run over Server-Sent Events, sessions use HTTP-only, SameSite=None, Secure cookies, and CORS is scoped to the ecosystem's own domains.
